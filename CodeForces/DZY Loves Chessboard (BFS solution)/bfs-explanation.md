# DZY Solution with BFS
Here's how I constructed this code and my thought process (primarily focusing on how I did the BFS since the rest is already done). Firstly, since recursion is a stack, we should define a queue of an int pair (since we need coordinates instead of just one integer). Then, we'll push the starting coordinates passed to the BFS function, because our BFS will start from this point on the grid. We then define a while loop that loops until the queue is empty, and within it create a loop that runs until 4 (standing for the 4 possible directions you can go on the grid). Then, we simply do the preliminary checks for the new coordinates (is it visited, out of bounds, etcetera), change the color, mark as visited, and repeat until the grid is full.

## Time complexity
NM loop and BFS contribute mainly to the time complexity, which is known to be **O(NM + NM)**.
